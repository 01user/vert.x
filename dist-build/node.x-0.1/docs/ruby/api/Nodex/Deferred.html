Overview

Represents something that hasn’t happened yet, but will occur when the #execute method is called. Once it has been executed it behaves identically to a Future.

Instances of Deferred can represent all kinds of deferred operations, e.g. copying a file or getting a value from a Redis server. Since the actual execution of the action is deferred until the execute method is called, it allows multiple instances of Deferred to be composed together into more complex control flows, e.g. using the orgorg.nodexorg.nodex.javaorg.nodex.java.coreorg.nodex.java.core.compositionorg.nodex.java.core.composition.Composer class.
